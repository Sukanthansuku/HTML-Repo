TYPE SELECTORS

  • Type selectors can be namespaced when using @namespace. This is useful when dealing with documents containing multiple namespaces such as HTML with inline SVG or MathML, or XML that mixes multiple vocabularies.
  • A Type Selector (sometimes referred to as an Element Type Selector) matches elements with the corresponding element node name, such as tags.
  • Often times Type Selectors are set as defaults, such as in a CSS reset where the intention is to override the browser defaults.
  • The Type Selectors above are setting the display property of those tags to block, so that any time any of those tags are used throughout the site they will display as block unless overwritten by a more specific style.
  • However, with a Type Selector such as body { }, setting a default font-size and line-height is common.
  • This is due in part to the fact that there can only be one body tag on any given page, so there are fewer opportunities for conflicts.
